# シフト作成アプリ「Shifty」企画表

## ■ Shifty サービス概要

Shiftyは、シフト作成作業を効率化し、管理者の負担を軽減するためのWebサービスです。Excelでの管理や複数ファイルの共有・比較といった煩雑な作業をなくし、半月ごとのシフト登録・編集・確定といった運用を一元化します。手作業で発生していた転記や確認業務を減らし、シフト作成にかかる時間を短縮することを目的としています。

## ■想定されるユーザー層

### ① 小規模店舗の店長・管理者

- 飲食店、カフェ、居酒屋、バーなど
- Excelでシフト調整していることが多く、運営作業が属人化しがち

### ② パート・アルバイトを複数抱える現場責任者

- スタッフの出勤希望や調整を毎月行っている
- シフト作成に時間がかかる

### ③ 個人経営・小規模チェーンの経営者

- システム導入するほどではないが、Excel管理に限界を感じている人

### ④ 社内の管理業務を兼任している人

- 店舗＋本部兼任（小規模企業に多い）
- シフト表の共有や確認業務で毎回時間を取られている

# ■ Shiftyの利用イメージ

Shiftyを使うと、管理者は複数のExcelやファイルを比較しながら作業する必要がなくなります。

ログイン後、半月分のシフト表を画面に表示し、クリック操作で勤務時間を登録・編集できます。作成が完了したら確定操作を1クリックで完了できます。これまで手作業で行っていた転記、ファイル共有、確認作業を減らし、シフト作成にかかっていた時間を短縮できるため、日常業務の負荷が軽減されます。

# ■ 利用手順

### ① Shiftyにログインする

管理者専用のアカウントでログイン。

### ② 半月分のシフト表を開く

スタッフ × 日付のシンプルな表が表示される。

### ③ シフトを登録する

該当スタッフのセルをクリックし、時間や休みを入力する。

### ④ 編集・調整を行う

追加・変更が必要な日をクリックするだけで修正可能。

### ⑤ シフトを確定する

問題がなければ「確定」を操作して完成。

## ■ ユーザー獲得方法について

- 無料テンプレ配布で集客：シフト表・休み希望表などを配布し、Excel管理の悩みを直接解決する導線を作る。
- SNS発信：XやInstagramで「シフトあるある」やUI動画を投稿し、日常的に困っている層へリーチ。
- 記事コンテンツ：noteやZennで「Excel管理の限界」など、課題・改善策を解説して検索流入を獲得。
- 実体験ベースの発信：現場改善ストーリーを公開し、課題共感で興味を引く。
- 無料小ツール提供：小規模な便利機能を先に公開し、Shiftyへ誘導する。
- 口コミ：知人や店舗へ直接紹介、1店舗の実績がそのまま宣伝になる。

# ■ Shifty 機能要件（MVP → 追加リリース順）

---

# ◆ Phase1 : **MVP（管理者だけで完結）**

目的：管理者のみでシフト作成できる最小構成を実装する。

### ● 認証・アカウント

- 管理者アカウント登録
- Laravel Sanctum によるSPA認証（管理者のみ）

### ● シフト作成・管理

- **半月ごとのシフト登録・編集**（クリック編集主体）
- 仮シフト保存
- **シフト確定ボタン**
- シフト状態管理（draft / confirmed）

### ● 表示・補助

- ガントチャート型のシフト表表示（DnDはまだ導入しない）
- 不足・過不足チェックや警告表示
- ポジション別のグルーピング表示
- 時間帯ごとの必要人数表示・強調

### ● データ基盤（DB）

- users、shifts、positions、required_headcounts、settings などの最低構成で実現

---

## ★ MVPで明確に採用しない（＝後回し）

- DnD
- PDF/外部連携
- スタッフアカウント
- 自動作成

# ◆ Phase2 : **スタッフアカウント導入**

目的：スタッフ機能を追加し、希望シフトの提出を可能にする。

### ● 認証

- 管理者によるスタッフ登録
- スタッフログイン（マルチガード）

### ● 希望シフト

- 希望入力期間の設定
- スタッフが希望を入力できる
- 未提出者の可視化
- 管理者が希望の取り込みができる

### ● 通知・利便性

- メール通知機能（前倒し検討の対象）

### ● DB側の主要追加

- shift_requests、notifications_settings の利用

---

---

# ◆ Phase3 : **高度化機能**

目的：操作性と機能を強化し、実用レベルに仕上げる。

### ● UI/UX強化

- **DnD対応（dnd-kit）**
- シフトテンプレ複製

### ● 拡張

- PDF出力
- Googleカレンダー／LINE連携
- **シフト自動作成**

# ■ 技術スタック・開発環境（Phase1〜想定）

Shiftyは、PHP/JavaScript構成を採用し、フロント・バックエンドを分離した構成で運用します。

Phase1では最小構成で実装し、将来的なスケールにも対応可能な選定としています。

---

## ◆ バックエンド

- **言語**：PHP 8.3
- **フレームワーク**：Laravel 12.x
- **認証**：Laravel Sanctum（SPA対応）
- **パッケージ管理**：Composer 2.x 系

---

## ◆ フロントエンド

- **ライブラリ**：React 19.x
- **言語**：TypeScript 5.9
- **ビルドツール**：Vite 5.x
- **UI**：Tailwind CSS 3.x
    - shadcn/ui、Radix UI を必要に応じて採用

---

## ◆ データベース／ストア

- **メインDB**：MySQL 8.0系（または MariaDB 10.11系）
- **キャッシュ／キュー**：Redis 7.x

---

## ◆ インフラ・実行環境

- Docker / Docker Compose
- Webサーバ：Nginx 1.26系
- Node.js：最新LTS（例：22 LTS 系）

---

## ◆ AWS構成（Phase1の最小構成を前提）

- **コンピュート**：Amazon EC2
    - Phase1では単体構成で十分（Laravel＋Reactビルド成果物をホスト）
    - 将来的には ECS/Fargate も検討可能
- **DB**：Amazon RDS for MySQL
- **ストレージ**：Amazon S3（ログ／バックアップ／静的ファイル）
- **DNS**：Amazon Route 53
- **監視・ログ**：CloudWatch Logs / Metrics

> ※PFの段階では「EC2単体構成」で十分現実的。将来的にスケールアップ可能な構成である点を明示。
>

---

- **VCS**：Git / GitHub
