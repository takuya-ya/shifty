# Git Workflow Rules

## コミットメッセージ

- コミットメッセージは必ず**日本語**で生成すること。
- 変更を行った場合は、提案するコミットメッセージをチャット欄にも必ず出力すること。
- **コミットはユーザーが行う**: `git commit` コマンドは実行しない。提案のみ行う。
- 形式:
  ```
  接頭辞: 本文
  - 理由の詳細（体言止め）
  ```
  （例）
  ```
  refactor: 未使用依存の除去とCSSテーマ変数をデフォルト命名に統一
  - 未使用依存の削除とshadcn/ui標準への統一
  ```

## PR タイトル

形式:
```
接頭辞: [タスク番号] タイトル本文
```
- 接頭辞は `feat` / `fix` / `refactor` / `chore` / `docs` などから選択。
- タスク番号はユーザーが明示した番号（例: `13.4`）を `[]` で囲む。番号が不明な場合はユーザーに確認すること。
- （例）`feat: [13.4] 管理者登録フォームを React Hook Form + Zod に移行`

## PR Description

- `.github/pull_request_template.md` のテンプレートに従って出力すること。
- 帰属フッター（「🤖 Generated with Claude Code」等）は追加しないこと。
